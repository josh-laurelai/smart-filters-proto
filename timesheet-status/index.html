Tradeoffs

✓ Pros Lowest implementation cost. Doesn't change page architecture or sorting. Released items naturally fade from view while staying accessible on hover. Blue = action, grey = done — color coding is clear.
✗ Cons Still requires users to scan the full list — no aggregation of "what needs my attention." If many entries are ready-to-release across many matters, users must scroll through everything. Progress bar helps but is small.
Variant B
Unreleased & Released Groups
Groups entries as Unreleased and Released within each matter. The Released section collapses to reduce noise. Within Unreleased, subtle inline cues differentiate errors from entries ready to release — no heavy color coding. Error entries show a small warning icon (⚠) next to the title. Ready-to-release entries show a subtle blue cue.

Tradeoffs

✓ Pros Clear hierarchy — errors and ready entries surface at the top, released entries collapse. Inline cues are subtle yet scannable. Fully-released matters collapse to one line. Reduces visual noise while keeping full context accessible.
✗ Cons Adds visual complexity within each matter card. Grouping may conflict with user's custom entry ordering. Collapse toggle requires a click to review released work.
Variant C
Action Summary Bar + Progressive Disclosure
A persistent summary bar surfaces what needs attention without requiring any scanning. Released entries are collapsed into a "show N released" toggle at the bottom of each matter. Matter cards get a left-border accent indicating their aggregate status.

Tradeoffs

✓ Pros Answers "what do I need to do?" instantly — no scanning required. Released entries are hidden by default, radically reducing visual noise. Left-border accents make matter-level status glanceable. "Release all" in the summary bar enables one-click batch action.

Variant D
Focus Tabs — Separate Views by Status
Replace the filter dropdown with persistent tabs that split the timesheet into focused views. "Needs Attention" becomes the default landing tab, showing only entries that require action. This is the most aggressive restructuring — it changes the mental model from "list of everything" to "show me what matters."
