Released entries genuinely disappear from your attention. Red stays for errors. No green anywhere — it's not needed.

Color System 1 — Tradeoffs

✓ Strengths Orange/amber is the strongest "act now" signal that isn't red. Grey is the most neutral possible — released entries genuinely vanish from attention. Variant B's collapsible Released section keeps the interface uncluttered. Clean, quiet design with no visual noise. Works well for colorblind users since it doesn't rely on red/green distinction.
✗ Risks Orange can read as "warning" rather than "ready" — users might hesitate to release because the color feels cautionary. Variant B requires users to expand Released section to see completed work. No positive reinforcement for released state (grey feels like absence rather than accomplishment).
Color System 2
Blue Action / Muted Sage
Blue = "this is your primary action" — same color as buttons and links. Users already associate blue with "click me." Muted sage/green-grey = "done" — a soft, earthy neutral that carries a subtle hint of "complete" without the vibrancy of full green. This system borrows from Jira's model where blue = active work.

Color System 2 — Tradeoffs

✓ Strengths Blue reuses the existing UI action color — users already know it means "do something." Sage is calm and mature without the vibrancy of green. Variant B's collapsible Released section keeps focus on actionable items. Jira precedent means design-savvy users already understand blue = active work.
✗ Risks Sage might feel too passive or cold for a "complete" state. Blue+Sage combination might not have enough contrast for some colorblind users. Orange for errors can clash slightly with blue. Variant B's collapsed view means users must expand to see all released items.
Color System 3
Green = Go / Purple = Done
Green = "GO, act on this now" — pure action color. Aligns with universal traffic-light intuition. Purple = "this is processed, inert" — unconventional for legal tools but distinctive. Borrows from GitHub's "merged" state, which is understood by millions of developers as "complete and archived." Red stays for errors.

Color System 3 — Tradeoffs

✓ Strengths Green aligns with universal traffic-light instinct — users immediately understand "GO, do this." Purple for "done" is novel and distinctive, preventing confusion with any action state. GitHub has normalized purple-for-merged with millions of developers. Variant B's collapsible Released section keeps focus on active work while archiving completed items.
✗ Risks Green for "ready" is a 180° semantic swap from current (where released used to be green) — may confuse existing users during transition. Purple for "released" is unconventional in legal/billing tools and may require user education. Some users may read purple as "special" rather than "inert." Variant B's collapsed view hides completed work from users who want to see their full history.
